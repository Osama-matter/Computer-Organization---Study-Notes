‚öñÔ∏è Comparisons

üî• SRAM vs. DRAM (Most Repeated)

SRAM (Static RAM)

  • Technology: Digital logic gates (flip-flops)
  • Refreshing: NO refreshing needed
  • Speed: Much faster
  • Size/Cost: Larger, more expensive
  • Usage: Cache Memory

DRAM (Dynamic RAM)

  • Technology: Stores data as charge on capacitors
  • Refreshing: Requires periodic refresh
  • Speed: Slower than SRAM
  • Size/Cost: Smaller, less expensive, higher density
  • Usage: Main Memory

üî• Hard Failure vs. Soft Error (2018, 2021, 2022)

Hard Failure

  • Nature: Permanent physical defect
  • Effect: Memory cell cannot store data correctly
  • Causes: Manufacturing defects, harsh environment, wear
  • Recovery: Requires hardware replacement

Soft Error

  • Nature: Random, non-destructive, temporary
  • Effect: Data altered, but memory not damaged
  • Causes: Power supply problems, alpha particles
  • Recovery: Can be corrected (ECC, rewriting)

üî• Write Through vs. Write Back (2019, 2021, 2023)

Write Through

  • Operation: Writes to both cache and main memory simultaneously
  • Advantage: Main memory always up to date
  • Disadvantage: More memory writes (slower)
  • Use Case: When data consistency is critical

Write Back

  • Operation: Writes only to cache, updates memory when replaced
  • Advantage: Minimizes memory writes, saves bandwidth
  • Disadvantage: Main memory may be stale
  • Use Case: Performance-critical applications

üìå SSD vs. HDD (2022)

HDD (Hard Disk Drive)

  • Mechanism: Rotating magnetic platters and mechanical heads
  • Speed: Slower (has seek time + rotational latency)
  • Durability: Vulnerable to physical shock
  • Noise: Makes noise due to moving parts

SSD (Solid State Drive)

  • Mechanism: Flash memory (no moving parts)
  • Speed: Much faster access time
  • Durability: More resistant to shock
  • Power: Lower power consumption, silent

üìå Batch Multiprogramming vs. Time Sharing (2018, 2022)

Batch Multiprogramming

  • Objective: Maximize processor use (Efficiency)
  • Directives: Job Control Language commands provided with job
  • User Interaction: Minimal or none

Time Sharing

  • Objective: Minimize response time
  • Directives: Commands entered at terminal by user
  • User Interaction: Interactive, real-time
