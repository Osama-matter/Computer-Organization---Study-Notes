6. Memory Systems

Memory Performance Metrics

Three Performance Measures:

Access Time

  • For Random Access Memory: Time it takes to perform a read or write operation
  • For Non-RAM: Time it takes to position the read/write mechanism at the desired location

Memory Cycle Time

Consists of the access time plus any additional time required after an operation.

7. Cache Memory

Elements of Cache Design

Cache: A small, fast memory that holds recently accessed data designed to speed up subsequent access to the same data.

1) Cache Address

  • Logical

2) Cache Size

3) Mapping Function

  • LRU (Least Recently Used)
  • FIFO (First In First Out)

  • Write Through
  • Write Back

Victim Cache

One approach to lower the miss penalty is to remember what was discarded in case it's needed again.

Four Most Common Replacement Algorithms:

1. LRU (Least Recently Used):

  • Most effective
  • Replace that block in the cache (longest without reference)

2. FIFO (First In - First Out):

  • Replace the block set in the cache (longest)
  • Easy implemented as a round robin of circular buffer technique

3. LFU (Least Frequently Used):

Replace the block that has experienced the fewest references.

11. Interrupts

Most Common Classes of Interrupts:
  • Program
  • Timer
  • I/O
  • Hardware Failure

Interrupt Processing

If an interrupt is pending, the processor does the following:

  1. It suspends the execution of the current program, save its context
  2. And do a interrupt handler routine

Two Approaches of Interrupts

Disable Interrupts:

When interrupts occurs, interrupts are then ignored or remain pending until a lower priority or interrupt routine complete. Interrupts are enabled before resuming the program.

Define Priorities:

Allow interrupt of a higher priority to cause a lower-priority interrupt handler to be itself interrupted.

12. Branch Prediction & Pipelining

Techniques to Improve Speed

Pipelining:

A processor can simultaneously work on multiple instructions.

Branch Prediction:

Processor looks ahead in the instruction code fetched from memory and predicts which branches is right.

Data Flow Analysis

Processor analyzes which instructions are depending on each other's result.

Speculative Execution

Using branch prediction and data flow analysis.

26. Memory Management

Types of Memory Management:

  • Swapping
  • Dynamic Partitioning
  • Logical Address: Expressed as location relative to the beginning of program
  • Physical Address: Actual location in main memory
  • Base Address: Currently starting location of the process

Virtual Memory (Demand Paging)

Each page of the process is brought when it is needed.

Advantages:

  • More processes can be maintained in memory
  • Time is saved because unused pages are not swapped in/out of memory

Disadvantages:

Thrashing: When the processor spends most of its time swapping pages rather than executing.
