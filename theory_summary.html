Chapter 1: Introduction

Computer Architecture vs Organization

Architecture: WHAT the computer does (instruction set, I/O, addressing)
Organization: HOW it does it (control signals, interfaces, memory technology)

Basic Components

  • CPU: Control Unit + ALU + Registers
  • Main Memory: RAM (volatile), ROM (non-volatile)
  • I/O Devices: Input/output peripherals
  • System Interconnection: Bus structures

Chapter 2: Evolution & Performance

Von Neumann Architecture:
  • Data and instructions in single read-write memory
  • Memory addressable by location
  • Sequential execution
  • Classified as SISD

Key Registers

Register Function
PC Program Counter - Address of next instruction
MAR/MBR Memory Address/Buffer Register
IR Instruction Register - 8-bit opcode
AC Accumulator - ALU operands/results

Chapter 4: Cache Memory

Cache Mapping Techniques:
  • Direct: i = j mod m (simple, can thrash)
  • Associative: Any line (flexible, expensive)
  • Set-Associative: Compromise (k-way)

Write Policies

Policy Description
Write Through Update cache AND memory (simple, more traffic)
Write Back Update cache only, write to memory when replaced (efficient)

Chapter 5: Internal Memory

SRAM vs DRAM

Feature SRAM DRAM
Technology Flip-flops Capacitors
Refresh Not required Required
Speed Faster Slower
Usage Cache Main memory

Chapter 6: External Memory

Disk Performance

Access Time = Seek Time + Rotational Latency

SSD vs HDD

Feature SSD HDD
Speed Much faster Slower
Mechanism Flash memory Rotating platters
Durability Shock resistant Vulnerable

Chapter 8: Operating System Support

OS Objectives:
  • Convenience: Easy to use
  • Efficiency: Efficient resource use

Scheduling Types

Type Function
Long-Term Admits programs to system
Short-Term (Dispatcher) Decides which ready process runs next
Medium-Term Manages swapping

Memory Management

  • Paging: Fixed-size frames, page table mapping
  • Segmentation: Variable-sized logical segments
  • Virtual Memory: Demand paging based on locality
  • TLB: Cache for page table entries
