Chapter 1: Introduction

Representation of Basic Information

Digital vs Analog Signals:
  • Analog signals: Continuous in nature
  • Digital signals: Discrete in nature

Computers are digital devices that work with two signal levels:

  • HIGH: Typically 5V or 12V (represented as 1)
  • LOW: Typically 0V (represented as 0)
Binary Number System: Uses symbols 0 and 1 to represent information and manipulate data in computers.
  • Bit: Binary digit (0 or 1)
  • Nibble: 4 bits
  • Byte: 8 bits

Computer Architecture vs Organization

Computer Architecture
  • Describes what the computer does
  • Functional behavior of system
  • Visible to programmers
  • Direct impact on program execution
  • Examples: instruction set, data representation, I/O mechanisms, addressing techniques
Computer Organization
  • Describes how it does it
  • Structural relationships
  • Hardware details transparent to programmers
  • Examples: control signals, interfaces, memory technology

System Description Levels

  1. Processor-Memory-Switch (PMS) level: Architect's view
  2. Instruction set level: Function of each instruction
  3. Register transfer level: Hardware structure visibility
  4. Logic gate level: Gates and flip-flops
  5. Circuit level: Resistors, transistors, capacitors, diodes
  6. Mask level: Silicon structures and IC layout

Basic Computer Functions

  • Data Processing: Computation and manipulation
  • Data Storage: Temporary and permanent storage
  • Data Movement: Input/output operations
  • Control: Managing all operations

Four Main Structural Components

  1. CPU: Central Processing Unit
    • Control Unit: Generates control signals
    • Arithmetic Logic Unit (ALU): Executes operations
    • Registers: Store data temporarily
  2. Main Memory: Primary storage (RAM, ROM)
  3. I/O Devices: Input and output peripherals
  4. System Interconnection: Bus structures

Memory Types

Volatile Memory

RAM (Random Access Memory): Loses data when power is off

Non-Volatile Memory
  • ROM (Read Only Memory)
  • PROM (Programmable ROM)
  • EPROM (Erasable PROM)
  • EEPROM (Electrically Erasable PROM)

Chapter 2: Computer Evolution and Performance

Von Neumann Machine

Von Neumann Architecture is based on three key concepts:
  1. Data and instructions stored in single read-write memory
  2. Memory contents addressable by location
  3. Execution occurs sequentially

IAS Computer Structure

Key Registers:
  • MBR (Memory Buffer Register): Contains word to be stored/received
  • MAR (Memory Address Register): Specifies memory address
  • IR (Instruction Register): Contains 8-bit opcode
  • IBR (Instruction Buffer Register): Temporarily holds right-hand instruction
  • PC (Program Counter): Address of next instruction
  • AC (Accumulator): Holds ALU operands/results
  • MQ (Multiplier Quotient): Holds operands/results

Microprocessor Speed Enhancement Techniques

  • Pipelining: Overlapping instruction execution
  • Branch Prediction: Predicting execution paths
  • Data Flow Analysis: Optimizing instruction scheduling
  • Speculative Execution: Executing ahead of time

Problems with Clock Speed and Logic Density

  1. Power: Increased power density and heat dissipation
  2. RC Delay: Wire resistance and capacitance limits
  3. Memory Latency: Memory speeds lag processor speeds

Improvement Approaches

  • Increase hardware speed of processor
  • Increase size and speed of caches
  • Change processor organization and architecture
  • Multicore approach: Multiple processors on same chip
Embedded System: Combination of computer hardware and software (and perhaps additional mechanical parts) designed to perform a dedicated function. Often part of a larger system (e.g., antilock braking system in a car).

Moore's Law

Number of transistors on a chip doubles every 18 months, leading to:

  • Decreased cost per transistor
  • Shortened electrical paths ‚Üí increased speed
  • Smaller physical size
  • Reduced power requirements
  • Fewer interchip connections

Chapter 3: Function and Interconnection

Instruction Cycle

Instruction Cycle: The processing required for a single instruction, consisting of:
  1. Fetch Cycle: Retrieve instruction from memory
  2. Execute Cycle: Perform the instruction

Instruction Cycle States

  • Instruction Address Calculation (iac): Determine next instruction address
  • Instruction Fetch (if): Read instruction from memory
  • Instruction Operation Decoding (iod): Analyze instruction
  • Operand Address Calculation (oac): Determine operand address
  • Operand Fetch (of): Fetch the operand
  • Data Operation (do): Perform the operation
  • Operand Store (os): Write result

Four Instruction Categories

  1. Processor-Memory: Data transfer between CPU and memory
  2. Processor-I/O: Data transfer between CPU and I/O
  3. Data Processing: Arithmetic and logic operations
  4. Control: Sequence alteration

Interrupts

Interrupt: A mechanism that allows other modules to interrupt normal processing, improving efficiency.

Common Interrupt Classes

Type Description
Program Errors in program execution
Timer Generated by processor's internal timer
I/O Signals from I/O controllers
Hardware Failure Power failure or memory parity error

Dealing with Multiple Interrupts

  1. Disable Interrupts: Process one interrupt at a time
  2. Define Priorities: Allow higher-priority interrupts to interrupt lower-priority handlers

Interconnection Structures

Bus Interconnection
  • Shared transmission medium
  • Multiple devices connect
  • Simple but can be bottleneck
Point-to-Point
  • Direct connections between pairs
  • Lower latency
  • Better scalability
  • Example: QPI, PCIe

Bus Structure

  • Data Bus: Transfers data (width determines bits per transfer)
  • Address Bus: Identifies source/destination (width determines max memory)
  • Control Bus: Controls access and timing
    • Memory read/write
    • I/O read/write
    • Bus request/grant
    • Interrupt signals
    • Clock and reset
PCI: Popular high-bandwidth, processor-independent bus
PCIe: Point-to-point interconnect replacing traditional bus architectures

Chapter 4: Cache Memory

Key Memory Characteristics

Characteristic Description
Location Internal (registers, cache, main memory) or External (peripherals)
Capacity Bytes for internal; varies for external
Unit of Transfer Number of bits transferred at once
Access Method Sequential, Direct, Random, Associative
Performance Access time, cycle time, transfer rate

Methods of Accessing Data

Sequential Access

Linear sequence; variable access time (e.g., tape)

Direct Access

Jump to vicinity then sequential; variable access time (e.g., disk)

Random Access

Unique physical address; constant access time (e.g., RAM)

Associative

Retrieved by content; constant time (e.g., cache)

Performance Parameters

  1. Access Time (Latency): Time to perform read/write operation
  2. Memory Cycle Time: Access time + recovery time before next access
  3. Transfer Rate: Rate of data transfer (1/cycle time for RAM)

Cache Memory Principles

Cache Memory: Small, fast memory between CPU and main memory that stores frequently accessed data to reduce average access time.
Principle of Locality:
  • Temporal Locality: Recently accessed items likely to be accessed again soon
  • Spatial Locality: Items near recently accessed items likely to be accessed

Cache Mapping Techniques

1. Direct Mapping

i = j modulo m

Where: i = cache line number, j = main memory block number, m = number of cache lines

  • Advantages: Simple, inexpensive
  • Disadvantages: Fixed location can cause thrashing

2. Associative Mapping

  • Any memory block can load into any cache line
  • Requires simultaneous examination of all tags
  • Advantages: Flexible, no thrashing
  • Disadvantages: Complex, expensive

3. Set-Associative Mapping

m = v √ó k, where i = j modulo v

Where: m = total lines, v = number of sets, k = lines per set (k-way associative)

  • Compromise between direct and associative
  • Block maps to specific set, can go in any line within set

Replacement Algorithms

  • LRU (Least Recently Used): Replace block unused longest - most effective
  • FIFO (First-In-First-Out): Replace oldest block - simple
  • LFU (Least Frequently Used): Replace least referenced block
  • Random: Random replacement

Write Policies

Write Through
  • All writes to cache AND main memory
  • Simple
  • Generates memory traffic
Write Back
  • Updates only in cache initially
  • Written to memory when replaced
  • Minimizes memory writes
  • More complex

Virtual Memory

Virtual Memory: Facility that allows programs to address memory from a logical point of view, without regard to physical memory available.
MMU (Memory Management Unit): Hardware module that translates each virtual address into a physical address in main memory.

Cache Types

  • Logical Cache (Virtual Cache): Stores data using virtual addresses
  • Physical Cache: Stores data using physical addresses
  • Unified Cache: Single cache for instructions and data
  • Split Cache: Separate instruction and data caches (eliminates contention)

Chapter 5: Internal Memory

DRAM vs SRAM

Characteristic DRAM SRAM
Storage Charge on capacitors Flip-flop logic gates
Refresh Required periodically Not required
Density Higher (smaller cells) Lower (larger cells)
Cost Less expensive More expensive
Speed Slower Faster
Usage Main memory Cache memory
Volatility Volatile Volatile

Read-Only Memory Types

  • ROM: Data wired during fabrication; cannot be changed
  • PROM (Programmable ROM): Written once electrically after fabrication
  • EPROM (Erasable PROM): Erased with UV light; multiple updates
  • EEPROM: Electrically erasable; byte-level updates; more expensive
  • Flash Memory: Block-level electrical erasure; between EPROM and EEPROM

Interleaved Memory

Interleaved Memory: Memory organized into multiple banks that can service requests simultaneously, increasing effective read/write rate by factor of K (number of banks).

Memory Errors

Hard Failure
  • Permanent physical defect
  • Cell stuck at 0 or 1
  • Causes: harsh environment, manufacturing defects, wear
Soft Error
  • Random, non-destructive event
  • No permanent damage
  • Causes: power problems, alpha particles

Error Correcting Codes (ECC)

Hamming Code: Can correct single-bit errors and detect double-bit errors
  • For M data bits, requires K check bits
  • K check bits where 2K ‚â• M + K + 1
  • Uses parity bits positioned at powers of 2
SEC-DED Code: Single-Error-Correcting, Double-Error-Detecting code - most common in semiconductor memory systems.

Advanced DRAM Types

  • SDRAM (Synchronous DRAM): Synchronized with system clock
  • DDR SDRAM (Double Data Rate): Transfers on both clock edges
  • RDRAM (Rambus DRAM): High-speed proprietary design
  • CDRAM (Cache DRAM): Integrates small SRAM cache

Chapter 6: External Memory

Magnetic Disk Organization

Key Terms:
  • Platter: Circular disk coated with magnetizable material
  • Track: Concentric rings of data on platter
  • Sector: Subdivision of a track; unit of data transfer
  • Cylinder: All tracks at same position on all platters
  • Gap: Separation between tracks or sectors

Disk Performance Parameters

  • Seek Time: Time to position head at desired track
  • Rotational Delay (Latency): Time for sector to rotate under head
  • Access Time: Seek time + rotational delay
  • Transfer Time: Time to transfer data once positioned
Access Time = Seek Time + Rotational Delay
Average Rotational Delay = 1/(2 √ó RPM/60)

Disk Types

  • Fixed-head: One read-write head per track
  • Movable-head: One head moves across tracks
  • Removable: Can be replaced (e.g., floppy, ZIP)
  • Non-removable: Permanently mounted (e.g., hard disk)

RAID (Redundant Array of Independent Disks)

RAID: Multiple disks viewed as single logical drive with data distribution (striping) and redundancy for reliability and performance.
RAID Level Description Advantages
RAID 0 Striping without redundancy High performance, no overhead
RAID 1 Mirroring (duplicate disks) High reliability, simple recovery
RAID 2 Bit-level striping with Hamming code Error correction
RAID 3 Byte-level striping with parity High transfer rate, single parity disk
RAID 4 Block-level striping with parity Independent disk access
RAID 5 Block-level with distributed parity No bottleneck, good performance
RAID 6 Block-level with dual parity Can survive two disk failures

Solid State Drives (SSD)

SSD: Memory device made with solid state components (flash memory) that can replace HDDs.

SSD vs HDD Comparison

Feature SSD HDD
Speed Much faster (no mechanical delay) Slower (seek time + rotational delay)
Durability More durable (no moving parts) Vulnerable to physical shock
Power Lower power consumption Higher power consumption
Noise Silent Audible spinning/seeking
Lifespan Limited write cycles Mechanical wear over time
Cost More expensive per GB Less expensive per GB

Flash Memory Types

NOR Flash
  • Basic unit: 1 bit
  • High-speed random access
  • Used for: BIOS, firmware
NAND Flash
  • Basic unit: 16 or 32 bits
  • Block-wise read/write
  • Used for: USB drives, SSDs, memory cards

SSD Practical Issues

  1. Performance Degradation: Slows down with use due to write amplification
  2. Limited Write Cycles: Flash cells wear out after certain number of writes
    • Mitigation: write caching, wear leveling, bad-block management

Optical Storage

  • CD-ROM: Read-only, 650-700 MB
  • CD-R: Write once, read many
  • CD-RW: Rewritable using phase change
  • DVD: Higher capacity (4.7-17 GB)
  • Blu-ray: High-definition, up to 50 GB dual-layer

Magnetic Tape

  • Sequential access medium
  • Parallel tracks running lengthwise
  • Blocks separated by inter-record gaps
  • Used for backup and archival storage
  • Very high capacity, low cost per GB

Chapter 8: Operating System Support

Operating System Objectives

Operating System: Program that controls execution of application programs and acts as interface between applications and hardware.
  • Convenience: Makes computer easier to use
  • Efficiency: Allows efficient use of system resources

OS Services

  • Program Creation: Utility programs and tools
  • Program Execution: Loading and running programs
  • Access to I/O Devices: Uniform interface to devices
  • Controlled Access to Files: File management and protection
  • System Access: Security and authentication
  • Error Detection and Response: Handling errors gracefully
  • Accounting: Usage statistics and billing

Key System Interfaces

  • ISA (Instruction Set Architecture): Machine language boundary between hardware and software
  • ABI (Application Binary Interface): Standard for binary portability; defines system calls
  • API (Application Programming Interface): High-level library calls for application portability
Kernel (Nucleus): The portion of OS in main memory containing most frequently used functions.

Types of Operating Systems

Dimension 1 Dimension 2
Batch System: Jobs batched and submitted by operator Uniprogramming: One program at a time
Interactive System: Direct user interaction with computer Multiprogramming: Multiple programs in memory; CPU switches among them

Desirable Hardware Features

  • Memory Protection: Prevent user programs from altering OS memory
  • Timer: Prevent jobs from monopolizing system
  • Privileged Instructions: Certain instructions only executable by OS
  • Interrupts: Allow OS to regain control flexibly

Scheduling Types

Type Description
Long-Term Decides which programs to admit to system (add to process pool)
Medium-Term Manages degree of multiprogramming (swapping decisions)
Short-Term (Dispatcher) Decides which ready process executes next; executes frequently
I/O Decides which pending I/O request to handle next

Five-State Process Model

  • New: Process just created
  • Ready: Waiting for CPU
  • Running: Currently executing
  • Blocked (Waiting): Waiting for I/O or event
  • Exit: Process terminated

Memory Management Techniques

Swapping

Swapping: Moving processes between main memory and disk to manage memory availability.

Partitioning

  • Fixed-size Partitions: Memory divided into fixed regions; simple but wasteful
  • Variable-size Partitions: Allocate exact amount needed; more efficient

Paging

Paging: Memory divided into fixed-size frames; process divided into pages of same size. Pages can be non-contiguous in memory.
  • Logical address = (page number, offset)
  • Physical address = (frame number, offset)
  • Page table maps page numbers to frame numbers

Segmentation

Segmentation: Memory viewed as collection of variable-sized segments. Visible to programmer; supports logical structure of programs.
  • Simplifies handling of growing data structures
  • Allows independent compilation and linking
  • Facilitates sharing and protection

Virtual Memory

Virtual Memory (Demand Paging): Pages brought into memory only when needed.
  • Based on principle of locality
  • Allows more processes in memory
  • Page fault triggers OS to load required page
TLB (Translation Lookaside Buffer): High-speed cache for page table entries; reduces address translation time.

Thrashing

Thrashing: Condition where processor spends most time swapping pages rather than executing instructions. Occurs when working set exceeds available memory.

Memory Management in Pentium II

  • Supports both segmentation and paging
  • Four modes:
    • Unsegmented unpaged (simple addressing)
    • Unsegmented paged (linear paging)
    • Segmented unpaged (logical partitions)
    • Segmented paged (both mechanisms)
  • Virtual address: 16-bit segment + 32-bit offset = 64 TB virtual space
  • Physical address: 32-bit = 4 GB physical space
  • Four privilege levels (0 = most protected)

ARM Memory Management

  • Two-level translation tables
  • Supports sections (1 MB), supersections (16 MB), large pages (64 KB), small pages (4 KB)
  • 16 domains for access control
  • Access permissions: no access, read-only, read-write
  • Privileged vs. unprivileged access
